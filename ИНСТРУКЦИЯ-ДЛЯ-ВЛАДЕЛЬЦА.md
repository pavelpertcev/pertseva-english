# Инструкция: как обновлять сайт через GitHub

Пошаговое руководство для владельца сайта Перцевой О.В.  
Сайт размещён на **GitHub Pages** (бесплатный хостинг от GitHub).  
Программы на компьютере устанавливать **не нужно** — всё делается в браузере на сайте [github.com](https://github.com).

**Адрес сайта:** [https://pertseva-english.ru](https://pertseva-english.ru)

---

## 1. Что нужно заранее

1. **Аккаунт на GitHub** — зарегистрируйтесь на [github.com/signup](https://github.com/signup), если аккаунта ещё нет.
2. **Приглашение в репозиторий** — помощник добавит ваш email или логин GitHub в настройки сайта. На почту придёт письмо «Invitation» — нажмите **Accept invitation**.
3. **Ссылка на репозиторий** — сохраните её в закладках. Пример адреса:

   `https://github.com/ЛОГИН/pertceva-site`

   (точный адрес даст тот, кто настраивал сайт)

После принятия приглашения вы сможете менять файлы сайта. Изменения появятся на сайте через **1–3 минуты** после сохранения.

### Важно: репозиторий публичный

Репозиторий сайта открыт для просмотра **любому пользователю GitHub** (код, учебные файлы, фото).  
**Редактировать** файлы могут только вы и те, кому выдан доступ (Collaborator).  
Сам сайт в браузере тоже публичный — так и задумано для учебных материалов.

---

## 2. Основные понятия (коротко)

| Слово | Что это значит |
|-------|----------------|
| **Репозиторий** | Папка с файлами сайта на GitHub |
| **GitHub Pages** | Сервис GitHub, который показывает сайт в интернете по адресу pertseva-english.ru |
| **Commit** («Сохранить изменения») | Кнопка, которая публикует ваши правки на сайт |
| **Edit (карандаш)** | Режим редактирования файла в браузере |

**Важно:** пока не нажали зелёную кнопку **Commit changes**, сайт **не обновится**.

---

## 3. Как сохранить любые изменения

После редактирования или загрузки файлов GitHub покажет форму внизу страницы:

1. В поле **Commit message** напишите коротко, что сделали, например:  
   `Добавлен КИМ по экономике 2026` или `Новая викторина Cinema 2`
2. Оставьте **Commit directly to the master branch** (или **main** — как называется основная ветка в репозитории)
3. Нажмите **Commit changes** (зелёная кнопка)

Готово. Подождите **1–3 минуты**, пока GitHub Pages обновит сайт. Затем откройте [pertseva-english.ru](https://pertseva-english.ru) и нажмите **Ctrl+F5** (полное обновление страницы).

## 4. Добавить новый учебный материал (файл Word)

Материал появится на сайте в **два шага**: сначала загрузить файл, потом добавить запись в каталог.

### Шаг 1 — загрузить файл

1. Откройте репозиторий на GitHub.
2. Зайдите в папку **`materials`** → выберите нужную подпапку:

   | Подпапка | Для чего |
   |----------|----------|
   | `Контрольно-оценочные материалы` | КИМ, КОС, ФОС |
   | `Методические разработки` | МР, СР, УП, статьи |
   | `Рабочие программы` | РП и программы дисциплин |
   | `Архив` | старые материалы (раздел «Архив» на сайте) |

3. Нажмите **Add file** → **Upload files**.
4. Перетащите файл (`.docx` или `.doc`) в окно или нажмите **choose your files**.
5. Внизу — **Commit message** (например: `Файл КИМ 2026`) → **Commit changes**.

**Совет:** имя файла лучше делать понятным, как у других файлов в папке, например:  
`КИМ — 38.02.01 Экономика — 2 курс (2026).docx`

### Шаг 2 — добавить материал в каталог

1. В корне репозитория откройте файл **`js/materials.js`**.
2. Нажмите иконку **карандаша** (Edit this file).
3. Найдите **самый верх** массива — сразу после строки `window.MATERIALS = [`.
4. **Скопируйте** блок ниже, вставьте **первым** в списке (перед остальными материалами).
5. Замените значения в **КАВЫЧКах** на свои (см. таблицу ниже).
6. **Commit changes**.

После сохранения материал **сам появится** на странице «Материалы», а счётчик **«учебных материалов»** на главной обновится (архивные материалы в счётчик не входят).

#### Шаблон записи (скопируйте и отредактируйте)

```js
  {
    id: 130,
    title: "КИМ — 38.02.01 Экономика — 2 курс (2026)",
    description: "Контрольно-измерительные материалы",
    category: "КИМ",
    section: "kom",
    date: "27.06.2026",
    downloads: [
      {
        label: "Скачать DOCX",
        url: "materials/Контрольно-оценочные материалы/КИМ — 38.02.01 Экономика — 2 курс (2026).docx",
      },
    ],
  },
```

После вставки **обязательно** должна быть **запятая** в конце блока `},` — как в примере.

#### Что менять в шаблоне

| Поле | Что писать |
|------|------------|
| **id** | Новое число, больше всех остальных (смотрите существующие id в файле; сейчас используются до ~130). Каждый материал — **уникальный** id. |
| **title** | Название на сайте (как видят студенты) |
| **description** | Короткое описание (1 строка) |
| **category** | Тип материала — см. таблицу категорий ниже |
| **section** | Раздел на странице «Материалы» — см. таблицу разделов |
| **date** | Дата в формате **ДД.ММ.ГГГГ**, например `27.06.2026` |
| **downloads → url** | Путь к файлу — **точно** как имя загруженного файла, включая папку |
| **downloads → label** | `Скачать DOCX` или `Скачать DOC` — в зависимости от типа файла |

#### Разделы (`section`)

| Значение | Где показывается на сайте | Папка для файла |
|----------|---------------------------|-----------------|
| `kom` | Контрольно-оценочные материалы | `materials/Контрольно-оценочные материалы/` |
| `methods` | Методические разработки | `materials/Методические разработки/` |
| `programs` | Рабочие программы | `materials/Рабочие программы/` |
| `archive` | Архив | `materials/Архив/` |

#### Категории (`category`) — примеры

| category | Когда использовать |
|----------|-------------------|
| `КИМ` | Контрольно-измерительные материалы |
| `КОС` | Контрольно-оценочные средства |
| `ФОС` | Фонд оценочных средств |
| `МР` | Методическая разработка |
| `СР` | Самостоятельная работа |
| `УП` | Учебное пособие |
| `РП` | Рабочая программа |
| `Статья` | Статья |
| `УМК` | Учебно-методический комплект (чаще в архиве) |

Путь в **downloads → url** должен **буква в букву** совпадать с именем файла на GitHub (включая пробелы, тире и русские буквы).

## 5. Добавить новую викторину (ссылка на MyQuiz)

Викторины хранятся в файле **`js/quizzes.js`** — так же, как материалы в `materials.js`.

1. Откройте **`js/quizzes.js`** → **Edit** (карандаш).
2. Найдите массив `window.QUIZZES = [`.
3. **Скопируйте** блок ниже и вставьте **в конец списка** (перед закрывающей `];`).
4. Замените значения в кавычках на свои.
5. **Commit changes**.

После сохранения викторина **сама появится** на странице «Викторины», а счётчик **«интерактивных викторин»** на главной обновится.

#### Шаблон записи викторины

```js
  {
    href: "https://play.myquiz.ru/p/XXXXXXXX",
    badge: "Грамматика",
    title: "Название викторины",
    description: "Краткое описание для студентов — о чём викторина, для какого курса",
  },
```

| Поле | Что писать |
|------|------------|
| **href** | Ссылка из MyQuiz: «Поделиться» → URL вида `https://play.myquiz.ru/p/00747151` |
| **badge** | Тема на карточке: `Грамматика`, `Лексика`, `Культура`, `Праздники` и т.п. |
| **title** | Заголовок викторины |
| **description** | Описание (1–2 предложения) |

После вставки **обязательно** должна быть **запятая** после `},` — кроме последней записи в списке.

Файл **`quizzes.html`** редактировать **не нужно** — карточки на странице строятся автоматически из `quizzes.js`.

## 6. Добавить сертификат (страница «Обо мне»)

Сертификаты показываются в галерее на странице **Обо мне**.

### Шаг 1 — загрузить изображение

1. Откройте папку **`certificates`** в репозитории.
2. **Add file** → **Upload files** → загрузите файл `.jpg`.
3. Имя файла — как у других: `Сертификат — краткое описание (год).jpg`
4. **Commit changes**.

### Шаг 2 — добавить запись в каталог

1. Откройте **`js/certificates.js`** → **Edit**.
2. Вставьте блок **в начало** массива `window.CERTIFICATES = [`:

```js
  {
    title: "Сертификат — краткое описание (2026)",
    file: "certificates/Сертификат — краткое описание (2026).jpg",
    date: "2026-06-19",
  },
```

| Поле | Что писать |
|------|------------|
| **title** | Подпись под превью (обычно совпадает с именем файла без `.jpg`) |
| **file** | Путь к файлу: `certificates/` + точное имя файла |
| **date** | Дата в формате `ГГГГ-ММ-ДД` (для сортировки) |

3. **Commit changes**.

## 7. Изменить email или другие настройки

Файл **`js/config.js`**:

- **contactEmail** — email для кнопки «Написать на электронную почту»

Редактируйте только значения **внутри кавычек**. После правки — **Commit changes**.

## 8. Краткая шпаргалка

| Задача | Файл / папка | Действие |
|--------|--------------|----------|
| Загрузить Word-файл | `materials/...` | Add file → Upload files |
| Показать материал на сайте | `js/materials.js` | Edit → вставить блок → Commit |
| Добавить викторину | `js/quizzes.js` | Edit → вставить блок → Commit |
| Email обратной связи | `js/config.js` | Edit → contactEmail → Commit |
| Добавить сертификат | `certificates/` + `js/certificates.js` | Загрузить JPG → добавить запись |

**После каждого изменения:** Commit message → **Commit changes** → подождать 1–3 минуты → **Ctrl+F5** на [pertseva-english.ru](https://pertseva-english.ru).
